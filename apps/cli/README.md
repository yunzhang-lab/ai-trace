# apps/cli

这里放 CLI、脚本、后台逻辑、适配层和共享工具。

## 核心原则

为了保证代码可以开源演进，同时不泄露任何私有数据，本目录遵循以下设计哲学：

- **全量开源，配置隔离**：所有处理逻辑都放在公开仓；私有工作区不存放核心处理代码。
- **零硬编码（Zero Hardcoding）**：代码里不允许出现个人绝对路径或敏感凭证。
- **动态传参执行**：代码必须通过环境变量或命令行参数接收工作区路径、输出路径和运行模式。
- **跨区执行**：当需要在私有仓处理数据时，是在私有仓上下文中调用公开仓工具。

公开仓是“公用工具箱”，私有仓是“个人加工台”。我们先把工具打磨好，再在私有区调用。

## 当前代码层级

`apps/cli/` 只保留一套代码树，内部按职责分层，不再把脚本和后台逻辑散在多个顶层目录：

- `scripts/`：CLI 脚本、ETL、扫描、导出、批处理任务。
- `backend/`：后台逻辑、状态变更、队列消费、动作执行。
- `adapters/`：公开仓与私有区之间的导入/导出映射层。
- `shared/`：通用函数、Schema、路径/文本归一化工具。
- `examples/`：示例输入输出与演示代码。
- `tests/`：后续如需补测试，再放这里。

根目录如保留少量兼容入口，也只作为过渡层使用；真正的实现应逐步迁入上述分层目录。

## 当前阶段的目标

当前阶段只保留：

- 可复用的脚本骨架
- 可说明的后台边界
- 可演示的导入示例
- 可复用的适配层

## 当前入口

- 统一 CLI 入口：`python3 apps/cli/main.py <command>`
- 当前首个命令：`python3 apps/cli/main.py scan`

后续如果要封装成 skill，优先从 `scripts/` 和 `adapters/` 中抽取稳定动作，再逐步上升为 `backend/` 的服务或任务编排。
