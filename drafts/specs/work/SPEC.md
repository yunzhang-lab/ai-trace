# Project 规范 (SPEC)

> 这是从 `CONTENT_MODEL_SPEC.md` 中拆解出的子规范。

## 1. 定位

项目 (`Project`) 默认公有。它不存储项目源码，而是存储项目的状态、进度、决策树和 Agent 参与情况。

## 2. 必须包含的共享信息

共享区（`work/projects/`）不一定保存完整的实体文件，但必须维护以下核心信息：

- **项目索引**：项目的全局唯一标识
- **真实路径**：指向用户硬盘上真实开发目录的指针
- **当前状态**：进行中、已暂停、已归档等
- **负责人**：主导该项目的实体（人或某个主控 Agent）
- **参与 Agent**：当前在该项目中活跃的 Agent 列表
- **决策 (Decisions)**：关键架构或业务决策的索引链接
- **交接 (Handoffs)**：最新的状态交接摘要链接

## 3. 存储建议

- 推荐使用 `.json` 或带有 YAML Frontmatter 的 Markdown 维护。
- `Project` 对象应当作为其他对象的聚合根（例如一个 `Decision` 应当关联到一个 `Project` ID）。
