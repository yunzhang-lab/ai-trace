## 2. 全局通用边界

| 项目 | 当前口径 |
| --- | --- |
| 真相源 | 真实治理层在 `~/.ai-trace`，H5 不是最终真相源 |
| 原始日志 | 原始对话、原始工具输出先留各 Agent 私有原生区 |
| 公私边界 | 公开仓不保存真实私有数据、敏感路径、原始会话原文 |
| 规范文件 | 正式总纲与公开 SPEC 默认公有；未确认内容留在 `drafts/`，确认后进入 `skill/ai-trace/specs/` |
| 数据流 | 采用“线性主干 + 受控分支 + 审计回流” |
| 数据载体 | 强结构对象倾向 `JSON / JSONL / SQLite`；语义沉淀内容倾向 `Markdown / Wiki` |
| 双仓阶段 | 公开仓与私有仓共用同一套阶段主轴，不各自再造独立路线 |
| 私有仓原则 | 私有仓只先定一级原则，具体目录结构按阶段增量建立 |
| Skill 原则 | Skill 先按单编排能力随阶段增长，稳定后再拆成多 Skill |

## 3. 通用状态与范围

### 3.1 `scope`

| 取值 | 说明 |
| --- | --- |
| `private` | 仅存在私有区 |
| `shared` | 可进入公开共享层 |
| `hybrid` | 同时有私有与共享表现形式 |

### 3.2 `status`

| 取值 | 说明 |
| --- | --- |
| `draft` | 草稿 |
| `candidate` | 候选 |
| `approved` | 已批准 |
| `rejected` | 已拒绝 |
| `resolved` | 已解决 |
| `merged` | 已归并 |
| `archived` | 已归档 |

## 5. 通用动作原则

### 5.1 提炼与归并

- 原始日志不是共享真相源
- 候选对象必须保留来源引用
- 多来源归并后，原对象不能直接消失，必须保留追溯链
- 不确定内容先保留候选或冲突，不直接覆盖稳定对象

### 5.2 审核与同步

- 审核动作至少包括：`approve / reject / private / merge`
- H5 可以发起动作，但不直接改写最终真相源
- 真正落盘由后端或主控执行层完成
- 所有关键动作必须追加写入审计链

### 5.3 审计

- 审计记录必须可追溯到对象、操作者、动作与时间
- 合并、覆盖、废弃、回滚都应保留留痕
- 审计是回流机制，不是可选附加项
